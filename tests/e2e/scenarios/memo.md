# 메모 앱 E2E 테스트 시나리오

## 1. 기능 영역: 메모 관리

### 1.1. 새 메모 작성 (성공)

- **시나리오 ID**: MEMO-001
- **우선순위**: Critical
- **목적**: 사용자가 필수 정보를 모두 입력하여 새 메모를 성공적으로 생성할 수 있는지 확인한다.
- **전제조건**: 사용자는 메인 페이지에 접속해 있다.
- **테스트 단계**:
    1. '새 메모' 버튼을 클릭한다.
    2. '새 메모 작성' 폼이 나타나는지 확인한다.
    3. '제목' 필드에 "새로운 테스트 메모"를 입력한다.
    4. '카테고리' 드롭다운에서 '아이디어'를 선택한다.
    5. '내용' 필드에 "이것은 새로운 테스트 메모의 내용입니다."라고 입력한다.
    6. '태그' 입력 필드에 "테스트"를 입력하고 '추가' 버튼을 클릭한다.
    7. '저장하기' 버튼을 클릭한다.
- **예상 결과**:
    - '새 메모 작성' 폼이 닫힌다.
    - 메모 목록에 "새로운 테스트 메모"가 나타난다.
    - 해당 메모에 '아이디어' 카테고리와 '#테스트' 태그가 표시된다.
- **실패 조건**:
    - '저장하기' 버튼을 클릭해도 폼이 닫히지 않는다.
    - 메모 목록에 새 메모가 추가되지 않는다.
    - 추가된 메모의 정보가 입력한 내용과 다르다.

### 1.2. 새 메모 작성 (필수 값 누락)

- **시나리오 ID**: MEMO-002
- **우선순위**: High
- **목적**: 필수 필드(제목, 내용)를 입력하지 않고 메모를 저장하려고 할 때, 적절한 오류 메시지가 표시되는지 확인한다.
- **전제조건**: 사용자는 '새 메모 작성' 폼을 열어두고 있다.
- **테스트 단계**:
    1. '제목' 필드를 비워둔다.
    2. '내용' 필드에 "내용만 있는 메모"라고 입력한다.
    3. '저장하기' 버튼을 클릭한다.
    4. '제목' 필드에 "제목만 있는 메모"를 입력한다.
    5. '내용' 필드를 비워둔다.
    6. '저장하기' 버튼을 클릭한다.
- **예상 결과**:
    - 각 단계에서 '저장하기'를 클릭했을 때, "제목을 입력해주세요." 또는 "내용을 입력해주세요."와 같은 유효성 검사 오류 메시지가 나타난다.
    - 메모가 생성되지 않고, 폼은 계속 열려 있다.
- **실패 조건**:
    - 오류 메시지 없이 메모가 저장된다.
    - 오류 메시지가 표시되지 않거나, 내용이 명확하지 않다.

### 1.3. 메모 검색

- **시나리오 ID**: MEMO-003
- **우선순위**: High
- **목적**: 사용자가 검색창에 키워드를 입력하여 원하는 메모를 찾을 수 있는지 확인한다.
- **전제조건**: 여러 개의 메모가 생성되어 있다.
- **테스트 단계**:
    1. 검색창에 "테스트"를 입력한다.
- **예상 결과**:
    - 제목이나 내용, 태그에 "테스트"가 포함된 메모만 목록에 표시된다.
    - 검색 결과와 일치하는 메모가 없을 경우, "검색 결과가 없습니다."와 같은 메시지가 표시된다.
- **실패 조건**:
    - 검색 결과가 입력한 키워드와 관련 없는 메모를 포함한다.
    - 검색 결과가 정확하지 않다.

### 1.4. 메모 필터링 (카테고리)

- **시나리오 ID**: MEMO-004
- **우선순위**: High
- **목적**: 사용자가 카테고리 필터를 사용하여 특정 카테고리의 메모만 볼 수 있는지 확인한다.
- **전제조건**: 다양한 카테고리의 메모가 생성되어 있다.
- **테스트 단계**:
    1. 카테고리 드롭다운 메뉴를 클릭한다.
    2. '업무' 카테고리를 선택한다.
- **예상 결과**:
    - '업무' 카테고리로 지정된 메모만 목록에 표시된다.
- **실패 조건**:
    - 다른 카테고리의 메모가 목록에 포함된다.
    - 필터링이 정상적으로 동작하지 않는다.

### 1.5. 메모 수정

- **시나리오 ID**: MEMO-005
- **우선순위**: High
- **목적**: 사용자가 기존 메모의 내용을 성공적으로 수정할 수 있는지 확인한다.
- **전제조건**: 수정할 메모가 최소 하나 이상 존재한다.
- **테스트 단계**:
    1. 특정 메모의 '편집' 버튼을 클릭한다.
    2. '메모 수정' 폼이 나타나고, 기존 내용이 채워져 있는지 확인한다.
    3. '제목'을 "수정된 테스트 메모"로 변경한다.
    4. '내용'에 "내용이 수정되었습니다."를 추가한다.
    5. '저장하기' 버튼을 클릭한다.
- **예상 결과**:
    - '메모 수정' 폼이 닫힌다.
    - 메모 목록에서 해당 메모의 제목과 내용이 수정된 정보로 업데이트된다.
- **실패 조건**:
    - 수정 사항이 저장되지 않는다.
    - 엉뚱한 메모가 수정된다.

### 1.6. 메모 삭제

- **시나리오 ID**: MEMO-006
- **우선순위**: Critical
- **목적**: 사용자가 메모를 영구적으로 삭제할 수 있는지 확인한다.
- **전제조건**: 삭제할 메모가 최소 하나 이상 존재한다.
- **테스트 단계**:
    1. 특정 메모의 '삭제' 버튼을 클릭한다.
    2. "정말로 삭제하시겠습니까?"와 같은 확인 대화상자가 나타나는지 확인한다.
    3. '확인' 버튼을 클릭한다.
- **예상 결과**:
    - 해당 메모가 메모 목록에서 사라진다.
- **실패 조건**:
    - 메모가 삭제되지 않는다.
    - 확인 대화상자 없이 즉시 삭제된다.
    - 엉뚱한 메모가 삭제된다.

## 2. 테스트 전략 요약

- **핵심 기능**: 메모의 CRUD(생성, 읽기, 수정, 삭제) 기능에 집중하여 테스트한다.
- **우선순위**:
    - **Critical**: 새 메모 작성, 메모 삭제
    - **High**: 메모 수정, 검색, 필터링, 필수 값 검증
- **커버리지**:
    - 주요 사용자 흐름(Happy Path)을 모두 포함한다.
    - 일반적인 오류 상황 및 엣지 케이스를 일부 포함한다. (e.g., 필수 값 누락)
- **향후 계획**:
    - 태그 관련 기능(추가, 삭제, 여러 태그 검색)에 대한 시나리오 추가
    - 마크다운 에디터 기능 테스트 시나리오 추가
    - 반응형 디자인 및 접근성 테스트 시나리오 추가
